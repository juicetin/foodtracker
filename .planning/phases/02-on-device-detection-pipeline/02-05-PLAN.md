1. Create `apps/mobile/src/screens/DetectionScreen.tsx`:

The main detection flow screen that orchestrates: photo selection -> spinner -> inference -> results display -> log meal.

Implementation:

**State machine flow:**
```
idle -> picking -> detecting -> results -> logging
```

- `idle`: Show "Take a Photo" or "Choose from Gallery" buttons (use expo-image-picker)
- `picking`: Image picker is open
- `detecting`: Show spinner with "Detecting foods..." text
  - Per locked decision: "Simple spinner with 'Detecting foods...' text during inference -- no animated scanning effect, no inference timing shown"
- `results`: Show AnnotatedPhoto + BoundingBoxOverlay + SummaryBar + DetectionList + ItemDetailSheet + LogMealFAB + UndoToast
  - Per locked decision: "Results persist on screen until user taps 'Log Meal' or dismisses -- no auto-timeout"
- `logging`: Save detected items to food_entries + ingredients tables via useFoodLogStore

**Photo selection:**
- Use expo-image-picker's launchCameraAsync() and launchImageLibraryAsync()
- Get photo URI, width, height from result
- Store in detection store via setPhoto()

**Inference flow (on photo selected):**
1. Set detecting state
2. Load models if not loaded (loadModelSet())
3. Preprocess image: resize to model input dimensions (640x640 for detection, 224x224 for binary/classify)
   - Use expo-image-manipulator or a simple resize approach
   - Convert to raw pixel buffer (Float32Array or Uint8Array) for model input
   - Note: This is the trickiest part. Start with expo-image-manipulator to resize, then read pixel data. If that's not feasible in Phase 2, use a placeholder that passes through and document as known limitation for Phase 2.5 or use a worklet.
4. Run runDetectionPipeline() with preprocessed image
5. For each detected item, call estimatePortion() from portionBridge to enrich with portion estimates
6. Set results in store via setItems()
7. Switch to results state

**Undo flow:**
- When item removed (via X button or swipe), show UndoToast
- Track last removed item ID
- On undo, call restoreItem() from store
- Auto-clear undo after toast timeout

**Detail sheet flow:**
- When item selected (bbox tap or list tap), set selectedItemId in store
- ItemDetailSheet opens for selected item
- Portion changes -> updatePortion in store
- Corrections -> correctItem in store + CorrectionStore.recordCorrection()

**Log Meal flow:**
- When LogMealFAB pressed:
  - Get active items from store
  - Create food entry in useFoodLogStore (or directly via userDb) with:
    - mealType from detection store
    - entryDate = today
    - Ingredients from active items (name, quantity=portionEstimate.weightG*multiplier, unit='g', aiConfidence, boundingBox coords)
    - Photo record linked to entry
  - Navigate back or show confirmation
  - Reset detection store

**Cross-highlight:**
- Per locked decision: "Tapping either box or list item opens the same detail bottom sheet; tapping one highlights the other to show connection"
- selectedItemId in store drives both BoundingBoxOverlay highlight and DetectionList highlight

**Layout:**
```
[AnnotatedPhoto with BoundingBoxOverlay]  ~50% screen height
[SummaryBar]                              ~40px
[DetectionList]                           remaining space (scrollable)
[LogMealFAB]                              absolute positioned bottom-right
[UndoToast]                               absolute positioned bottom-center
[ItemDetailSheet]                         bottom sheet overlay
```

**Image preprocessing note:**
The most pragmatic approach for Phase 2 is:
- Use expo-image-manipulator to resize the photo to model input dimensions
- For the pixel buffer conversion, check if react-native-fast-tflite accepts image URIs directly or requires raw buffers
- If raw buffers needed and no easy RN solution exists, create a simple helper that reads the resized image and converts to the expected format
- Document any preprocessing limitations as known issues for Phase 2
