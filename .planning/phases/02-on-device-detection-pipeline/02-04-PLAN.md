1. Create `apps/mobile/src/store/useDetectionStore.ts`:

Zustand store following project conventions (see useFoodLogStore.ts for pattern):

```typescript
interface DetectionStore extends DetectionSessionState {
  // Actions
  setPhoto: (uri: string, width: number, height: number) => void;
  setItems: (items: DetectedItem[]) => void;
  setDetecting: (detecting: boolean) => void;
  removeItem: (id: string) => void;
  restoreItem: (id: string) => void;
  updatePortion: (id: string, multiplier: number) => void;
  correctItem: (id: string, newClassName: string) => void;
  setMealType: (type: MealType) => void;
  selectItem: (id: string | null) => void;
  reset: () => void;
  // Computed
  activeItems: () => DetectedItem[]; // items where isRemoved === false
}
```

Key behaviors:
- `removeItem`: Sets isRemoved=true and removedAt=Date.now() (soft delete for undo). Does NOT persist to SQLite (detection session is ephemeral until "Log Meal").
- `restoreItem`: Sets isRemoved=false and removedAt=undefined.
- `updatePortion`: Sets portionMultiplier on the item (0.5-3.0 clamped).
- `correctItem`: Updates className and sets correctedFrom to original name.
- `activeItems`: Returns items.filter(i => !i.isRemoved), sorted by confidence descending (per locked decision).
- `setMealType`: Sets mealType directly (user override of autoDetectMealType).
- `reset`: Clears all state back to initial values.
- Initial mealType set via autoDetectMealType() on store creation.
